# Настройка локальной и WG-сети на базе роутера Keenetic

## Описание

Данный проект представляет собой пошаговое руководство по настройке безопасной домашней сети на базе роутера Keenetic с VPN-сервером WireGuard.

### Архитектура сети

Настройка создаёт три сетевые зоны:

1. **Keenetic-LAN** (например, 192.168.1.0/24) - локальная сеть за роутером Keenetic
2. **WG-LAN** (например, 10.0.0.0/24) - сеть WireGuard VPN для безопасного удалённого доступа
3. **OpenWRT-LAN** (например, 192.168.99.0/24) - вторичная локальная сеть с роутером OpenWRT в качестве клиента WG

### Ключевые принципы безопасности

- Web-интерфейс роутера доступен только из Keenetic-LAN и WG-LAN (не напрямую из интернета)
- SSH-доступ ограничен локальными сетями и WG-сетью
- Локальные сервисы (например, `srvok.ckox`) доступны через WG-туннель, но не напрямую из интернета
- Пользовательский DNS на Keenetic разрешает локальные доменные имена (например, `*.srvok.ckox`)

### Целевая платформа

- **Роутер:** Keenetic (любая модель с поддержкой WireGuard), проверка делалась на версии прошивки 4.3.6.3
- **Клиент PC:** Ubuntu 24.04 (все инструкции для Linux)
- **Дополнительный роутер:** OpenWRT (опционально)
- **Локальный сервер:** Ubuntu 24.04 (IP: 192.168.1.11)
- **Сервисы:** Устанавливаются в Docker контейнерах

## Важные замечания

### Безопасность

- Всегда создавайте резервные копии перед внесением изменений
- Не закрывайте текущую SSH-сессию, пока не убедитесь, что новая работает
- Запрещайте парольную аутентификацию только после проверки входа по ключу
- Храните приватные ключи в безопасном месте

### Сеть

- Убедитесь, что выбранные подсети не пересекаются
- Запишите все выбранные IP-адреса и порты
- Внешний IP роутера может меняться (настройте DDNS при необходимости)

### Docker

- Все сервисы на локальном сервере (192.168.1.11) устанавливаются в Docker контейнерах
- Это обеспечивает изоляцию и упрощает управление
