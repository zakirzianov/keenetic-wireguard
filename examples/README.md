# Примеры конфигураций

Эта директория содержит готовые к использованию конфигурационные файлы для настройки обратного прокси-сервера.

## Доступные конфигурации

### [Caddy](./caddy/) (Рекомендуется)

Простая и автоматизированная конфигурация с Caddy:
- Автоматическое перенаправление HTTP → HTTPS
- Простая конфигурация
- Встроенная поддержка HTTPS
- Меньше настроек

**Когда использовать:**
- Для быстрого старта
- Если нужна простая конфигурация
- Если не требуется тонкая настройка

### [Nginx](./nginx/) (Альтернатива)

Более гибкая конфигурация с Nginx:
- Явное управление редиректами
- Детальная настройка прокси
- Больше возможностей для оптимизации
- Более низкое потребление ресурсов

**Когда использовать:**
- Если нужен больший контроль
- Если уже есть опыт с Nginx
- Если требуется специфическая настройка

## Общие требования

Для обеих конфигураций необходимо:

1. **Docker и Docker Compose**
   ```bash
   sudo apt install docker.io docker-compose-plugin
   ```

2. **SSL-сертификаты (mkcert)**
   - Wildcard сертификат `*.srvok.ckox`
   - Установленный корневой CA на клиентах

3. **Настроенный DNS**
   - DNS-записи на роутере Keenetic
   - Wildcard запись `*.srvok.ckox → 192.168.1.11`

4. **Запущенные сервисы**
   - Тестовый web-сервер на порту 9898
   - NAS сервис на порту 9191

## Быстрый старт

### Вариант 1: Caddy (рекомендуется)

```bash
# Скопировать конфигурацию
cp -r examples/caddy ~/caddy
cd ~/caddy

# Создать директорию для сертификатов и сгенерировать их
mkdir certs && cd certs
mkcert "*.srvok.ckox" "srvok.ckox"
mv _wildcard.srvok.ckox+1.pem srvok.ckox.crt
mv _wildcard.srvok.ckox+1-key.pem srvok.ckox.key

# Запустить
cd ~/caddy
docker compose up -d
```

### Вариант 2: Nginx

```bash
# Скопировать конфигурацию
cp -r examples/nginx ~/nginx
cd ~/nginx

# Создать директорию для сертификатов и сгенерировать их
mkdir certs && cd certs
mkcert "*.srvok.ckox" "srvok.ckox"
mv _wildcard.srvok.ckox+1.pem srvok.ckox.crt
mv _wildcard.srvok.ckox+1-key.pem srvok.ckox.key

# Запустить
cd ~/nginx
docker compose up -d
```

## Проверка работы

После запуска проверьте доступность сервисов:

```bash
# Проверить главный сервис
curl https://srvok.ckox

# Проверить роутер через прокси
curl https://keenetic.srvok.ckox

# Проверить NAS сервис
curl https://nas.srvok.ckox
```

## Переключение между конфигурациями

Если вы решите переключиться с одной конфигурации на другую:

```bash
# Остановить текущую конфигурацию
docker compose down

# Скопировать сертификаты в новую конфигурацию
cp ~/caddy/certs/* ~/nginx/certs/
# или наоборот
cp ~/nginx/certs/* ~/caddy/certs/

# Запустить новую конфигурацию
cd ~/новая-конфигурация
docker compose up -d
```
