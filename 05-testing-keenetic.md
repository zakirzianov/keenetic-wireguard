# Шаг 5: Тестирование Keenetic

## Тестирование

[x] Проверить доступ к web-интерфейсу роутера из локальной сети

**На Ubuntu 24.04 (из локальной сети Keenetic-LAN):**

```bash
# Проверить доступ через браузер или curl
curl -I http://192.168.1.1

# Открыть в браузере
firefox http://192.168.1.1
```

**Ожидаемый результат:** Доступ должен быть разрешён, страница авторизации роутера должна открыться

[x] Проверить доступ по `ssh` из локальной сети

**На Ubuntu 24.04:**

```bash
# Подключиться к роутеру по SSH
ssh -p 2222 myuser@192.168.1.1

# Если подключение успешно - выйти
exit
```

**Ожидаемый результат:** SSH-соединение должно успешно установиться

[x] Проверить ping роутера из локальной сети

**На Ubuntu 24.04:**

```bash
# Проверить ping роутера
ping -c 4 192.168.1.1
```

**Ожидаемый результат:** Пакеты должны доходить, 0% потерь

[x] Проверить невозможность доступа к web-интерфейсу роутера из интернета напрямую

**Из внешней сети (или через мобильный интернет, отключив WiFi):**

```bash
# С внешнего устройства (мобильный интернет) попробовать подключиться
curl -I http://ВНЕШНИЙ_IP_РОУТЕРА --max-time 10
```

**Ожидаемый результат:** Соединение должно быть отклонено или истекать по таймауту

[x] Проверить невозможность доступа по `ssh` из интернета напрямую

**Из внешней сети:**

```bash
# Попробовать подключиться по SSH к внешнему IP
ssh -p 2222 myuser@ВНЕШНИЙ_IP_РОУТЕРА -o ConnectTimeout=10
```

**Ожидаемый результат:** Соединение должно быть отклонено

[x] Проверить доступность UDP-порта WireGuard из интернета (например, с помощью `nc` или онлайн-сервисов проверки портов)

**На Ubuntu 24.04 (из внешней сети):**

```bash
# Установить nmap (если не установлен)
sudo apt update && sudo apt install -y nmap

# Проверить UDP порт WireGuard
sudo nmap -sU -p 51820 ВНЕШНИЙ_IP_РОУТЕРА
```

**Через онлайн-сервисы:**

- Открыть <https://check-host.net/check-udp?host=ВНЕШНИЙ_IP_РОУТЕРА:51820>

**Ожидаемый результат:** Порт должен быть открыт (open/filtered)
