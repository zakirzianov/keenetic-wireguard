# Шаг 10: Разработка тестового скрипта

## Разработка скрипта для проверки корректности настроек

[ ] Выбрать язык/технологию для скрипта

**Рекомендация:** Bash (доступен на всех системах) или Python (более гибкий)

Для данного руководства используем **Bash** для максимальной совместимости.

[ ] Определить, откуда будет запускаться скрипт

**Рекомендация:** Универсальный скрипт, который можно запускать из любой сетевой зоны. Скрипт автоматически определит текущую зону и выполнит соответствующие проверки.

[ ] Написать скрипт для проверки всех сервисов

**Готовый скрипт:**

Скрипт доступен в файле [`network-test.sh`](./network-test.sh) в корне репозитория.

**Использование скрипта:**

```bash
# Скачать скрипт из репозитория (если еще не скачан)
# wget https://raw.githubusercontent.com/YOUR_REPO/keenetic-wg/main/network-test.sh

# Или скопировать из локального репозитория
cp network-test.sh ~/network-test.sh

# Сделать скрипт исполняемым
chmod +x ~/network-test.sh

# Запустить скрипт (для полной проверки WireGuard требуется sudo)
sudo ~/network-test.sh
```

**Настройка под вашу сеть:**

Перед использованием отредактируйте переменные конфигурации в начале скрипта:

```bash
# Конфигурация (настроить под вашу сеть)
KEENETIC_IP="192.168.1.1"      # IP-адрес роутера Keenetic
SERVER_IP="192.168.1.11"        # IP-адрес локального сервера
WG_SUBNET="10.0.0.0/24"         # Подсеть WireGuard
LAN_SUBNET="192.168.1.0/24"     # Подсеть локальной сети
```

**Возможности скрипта:**

Скрипт автоматически:
- Определяет текущую сетевую зону (Keenetic-LAN, WG-LAN, OpenWRT-LAN)
- Проверяет статус WireGuard (интерфейс, handshake, свежесть соединения)
- Тестирует доступность роутера (ping, HTTP, HTTPS, SSH)
- Проверяет локальный сервер и DNS-разрешение
- Тестирует HTTPS-сервисы и валидность SSL-сертификатов
- Выводит результаты в цветном формате с символами статуса

[ ] Скрипт должен автоматически определять сетевую зону

**Уже реализовано в функции `detect_network_zone()`**

Скрипт определяет зону по:
- Наличию и активности интерфейса wg0
- IP-адресу машины (192.168.1.x, 192.168.99.x, 10.0.0.x)

[ ] Скрипт должен выводить результат в удобном формате

**Уже реализовано:**
- Цветной вывод (зеленый ✓, красный ✗, желтый ?)
- Таблица с результатами
- Детали для каждого теста
- Итоговая сводка

[ ] Протестировать скрипт из всех сетевых зон

**Из Keenetic-LAN:**
```bash
cd ~/network-tests
./network-test.sh
```

**Из WG-LAN (с активным WG):**
```bash
# Подключиться к WG
sudo wg-quick up wg0

# Запустить скрипт
cd ~/network-tests
sudo ./network-test.sh
```

**Из OpenWRT-LAN:**
```bash
# Скопировать скрипт на устройство в OpenWRT-LAN
scp ~/network-tests/network-test.sh user@openwrt-device:~/

# Запустить на устройстве
ssh user@openwrt-device
chmod +x ~/network-test.sh
./network-test.sh
```

**Из интернета (без WG):**
```bash
# Отключить WG
sudo wg-quick down wg0

# Запустить скрипт (многие тесты должны провалиться - это ожидаемое поведение)
./network-test.sh
```

[ ] Убедиться, что скрипт корректно выявляет ошибки конфигурации

**Проверки, которые выполняет скрипт:**

1. **WireGuard:**
   - Наличие интерфейса wg0
   - Активность handshake
   - Свежесть handshake (< 3 минут)

2. **Роутер Keenetic:**
   - Доступность по ping
   - Доступность web-интерфейса (HTTP/HTTPS)
   - Доступность SSH

3. **Локальный сервер:**
   - Доступность по ping
   - Работа тестового web-сервера на порту 9898
   - DNS-разрешение srvok.ckox

4. **HTTPS сервисы:**
   - Доступность https://srvok.ckox
   - Доступность https://keenetic.srvok.ckox
   - Доступность https://nas.srvok.ckox
   - DNS-разрешение поддоменов

5. **SSL сертификаты:**
   - Валидность сертификатов
   - Отсутствие ошибок проверки

## Расширенные возможности

### Добавление логирования

Чтобы сохранять результаты тестов:

```bash
# Запустить с логированием
./network-test.sh | tee test-results-$(date +%Y%m%d-%H%M%S).log
```

### Автоматизация через cron

Для периодических проверок:

```bash
# Добавить в crontab
crontab -e

# Запускать каждый час
0 * * * * /home/user/network-tests/network-test.sh >> /home/user/network-tests/test.log 2>&1
```

### Интеграция с системой мониторинга

Скрипт можно интегрировать с:
- **Prometheus** + Node Exporter (экспорт метрик)
- **Grafana** (визуализация)
- **Nagios/Zabbix** (алертинг)

### Добавление уведомлений

Отправка результатов по email или в Telegram:

```bash
# Отправка по email при ошибках
./network-test.sh | mail -s "Network Test Results" admin@example.com

# Отправка в Telegram
# (требует telegram-send: pip install telegram-send)
./network-test.sh | telegram-send --stdin
```

## Дополнительные проверки

Можно добавить в скрипт:

1. **Проверка производительности:**
   - Скорость подключения через WG (iperf3)
   - Задержка (latency) до различных хостов
   - Пропускная способность

2. **Проверка безопасности:**
   - Сканирование открытых портов (nmap)
   - Проверка силы шифрования SSL
   - Проверка версий ПО на уязвимости

3. **Проверка доступности из интернета:**
   - Проверка внешнего IP
   - Проверка доступности порта WireGuard из вне
   - Проверка недоступности закрытых портов

4. **Мониторинг ресурсов:**
   - Загрузка CPU/RAM на роутере и сервере
   - Использование дискового пространства
   - Температура оборудования
